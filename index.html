HTML (Hypertext Markup Language)

  • Think of HTML as the bricks and structure of a house.
  • The <head> tag contains information about the webpage.
  • The <body> tag contains the content and the structure of the webpage.
  • HTML uses angel brackets (<>) for the tags.
  • When using HTML, try to use semantic coding. This will help who require assistance (such as screenreaders) to navigate your website.

CSS (Cascading Stylesheet)

  • Think of CSS as decoration for the house. The colours of the walls, socket faces, skirt etc.
  • When dealing with CSS, keep in mind the "box model". Everything in CSS has a box around it.
  • The "box model" layout:
    1. Start off with your content.
    2. Then padding is what adds space on the inside of the content box.
    3. Then comes a border which sits inbetween the padding and the margin.
    4. Then comes a margin that adds space on the outside of the content box.
  • Files end in ".css".
  • There are three ways to add CSS to a webpage:
    1. Inline: inline CSS is added to HTML elemant itself. It can "only effect" that elemant.
    2. Internal: internal CSS is created in the same document as your HTML but typically resides in the HEAD. This is also a bad practice since it can "only effect" the HTML in that document.
    3. External: external CSS is what is most commonly used. External CSS has it's own document and can be linked into as many HTML documents as you'd like.
  • CSS3 has a lot of new great features. One of them being variables! You can now add custom variables to your CSS.

GIT & GitHub

  • GIT isn't like the other topics in this guide. GIT is a version control system which lets you take snapshots of your project and fork off from that project.
  • GitHub is an online service used to store and hold your repositorys.
  • GIT & GitHub allows for many users to work and write on the same code/project at the same time!
  • GIT & GitHub are great for preventing the main from being broken.
  • GIT was created by the same person who created Linux, Linus Torvalds.
  • GIT operates on your computer via the terminal. You don't need to register for account, just download and install.
  • Here are a few of the commands and what they do:
    • git pull origin *branch* / pulls the lastest version of that branch from GitHub.
    • git checkout -b *branch* / creates a new branch.
    • git checkout *branch* or git switch *branch* / allows you to change what branch you're on in your repository.
    • git add -A / adds your branch to the processing area ready for you to commit.
    • git commit -m "your message" / commits the work in the repository and branch you're in ready to push to GitHub.
    • git push origin *branch* / pushes your work to GitHub.
    • git push origin --delete *branch* / deletes the branch from GitHub and your workflow.
    • git status / checks to see what branch your working on.
  • GitHub pages allows you to host your repository for free!

JS (JavaScript)

  • Look at JavaScript as if it's the functionality in an house. Doors that open and close, light being switched on or off, radiatores heating up your home etc.
  • JavaScript is going to be hardest topic to wrap your hear around since it's the only fully-fledged programming language in this guide.
  • JS can make decisions based on the code passed into it.
  • JS has a control flow that reads from top of the page to the bottom. This can be interupted a few different ways such as conditional statements and loops.
  • Conditional statements are a way we introduce decision making (like I mentioned above) into our code.
  • Variables are a container hat store data. An example would be a box called teddy which contains a stuffed animal inside.
  • Arrays are similar to variables but these can hold multiple data types at the same time and is mutable.
  • Loops are used to perform a task multiple times over.
  • Functions are blocks of code that do not run on their own. They have be called somewhere on the document and they can be called multiple times over.
  • Math.floor & Math.random are both properties of JavaScripts built in math object can can be used together to create a random number.
  • A script tage typically sits on the bottom of your code just before the </body> tag. This is so the page can render before the script can be ran. Running the script before the content is render can lead to things not loading correctly.
  • Like CSS, JS can be internal and external. For the same reasons as CSS, it's best practice to use an external script document which ends in .js.
